# RefundBot - Бот возвратов ДС

Telegram-бот для мониторинга ошибок в возвратах денежных средств из Google Таблицы.

## ✨ Особенности

- 🔄 Автоматическая проверка таблицы каждые 5 часов
- 📊 Двойное кеширование для отслеживания изменений
- 🎯 Фильтрация только по нашим ПВЗ (27 шт.)
- 📝 Команды для каждого статуса обработки
- 🚀 Retry механизм при ошибках сети
- 🕐 Часовой пояс Asia/Tashkent (UTC+5)

## 🚀 Деплой на Railway

1. Создайте репозиторий на GitHub:

```bash
cd RefundBot
git init
git add .
git commit -m "init refund bot"
git remote add origin https://github.com/YOUR_USERNAME/RefundBot.git
git push -u origin main
```

2. В Railway:
   - New Project → Deploy from GitHub repo
   - Выберите репозиторий
   - В разделе **Variables** добавьте:
     - `BOT_TOKEN`
     - `SPREADSHEET_ID`

3. Railway автоматически установит зависимости и запустит бота

## 📊 Статусы обработки

| Статус | Описание | Действие |
|--------|----------|----------|
| *Пустой* | Новая ошибка | Уведомление о появлении |
| Связались с клиентом, придет заполнять заявление | Передано в ПВЗ | Напоминание на следующий день |
| Клиент не выходит на связь | Не удается связаться | Напоминание на следующий день |
| Клиент отказывается дать данные добровольно | Отложено | Можно обработать позже |
| Обработали, заявление загружено в WMS | Завершено | Ошибка устранена |

## 🔔 Уведомления

Бот отправляет уведомления об изменениях:
- 🆕 **Новые ошибки** — появились в таблице
- ✅ **Устранены** — удалены из таблицы
- 🔄 **Изменены** — изменился статус или данные

**Двойное кеширование:**
- Первый запуск: все записи считаются новыми
- Второй запуск: сравнение с первым кешем, показ изменений
- Третий запуск: сравнение со вторым кешем, первый кеш удаляется

## 🌐 Google Sheets — важно!

Таблица должна быть **открыта для просмотра по ссылке**.

**Структура колонок:**
- A: Номер заказа (используется)
- B: ПВЗ
- C: Дата возврата
- D: Тип оплаты
- E: Сумма возврата
- F: Клиент
- G: Причина доработки
- H: Статус обработки
- I: Номер заказа (дубликат, игнорируется)

## 🛠 Технические детали

- **Retry механизм**: 3 попытки с экспоненциальной задержкой
- **Двойное кеширование**: отслеживание изменений между проверками
- **Фильтрация**: только наши 27 ПВЗ из 770+ строк
- **Форматирование**: длинные причины сокращаются до 100 символов
- **Доступ**: только владелец (ID: 6061065577)
